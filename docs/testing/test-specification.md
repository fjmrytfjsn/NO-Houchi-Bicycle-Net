# テスト仕様書

## 1. 目的

本書は、[基本設計仕様書](../design/basic-design.md) を主な参照元として、NO-Houchi Bicycle Net 試作品のテスト対象、テスト観点、確認内容、テスト条件、テスト手順、期待値を整理するものである。

テスト実行後に結果を追記できるよう、各テストケースに `実行日` `実行者` `実行結果` の記録欄を設ける。

## 2. 対象範囲

- サポーター用 Android アプリ
- 持ち主用 Web アプリ
- 管理者用ダッシュボード
- バックエンド API

以下は将来構想として扱い、本書の対象外とする。

- ブラックリスト管理
- 高度分析、ヒートマップ
- 回収後の保管、返還、処分管理
- クーポンの詳細発行、利用管理
- 警察、店舗などの外部システム本連携
- 撮影品質判定、ジオフェンシングの高度化

## 4. テストレベル定義

| テスト観点 | 定義 |
| --- | --- |
| 単体テスト | 1つの関数、API、画面部品、画面内ロジックを個別に確認するテスト |
| 結合テスト | 画面とAPI、APIとDB、画面遷移など、複数要素の連携を確認するテスト |
| システムテスト | 試作品の業務シナリオ全体が成立することを確認するテスト |

## 5. 実施記録ルール

- `実行日`: 実際にテストを実行した日付を記入する
- `実行者`: テスト実施者名を記入する
- `実行結果`: 初期値は `未実施` とし、実施後に `OK` `NG` `保留` などへ更新する

## 7. 主要シナリオ対応

| シナリオ | 対応する主なテストケース |
| --- | --- |
| 通報 → 持ち主が解除して解決 | AND-02, OWN-01, OWN-02, OWN-05, API-01, API-06, API-07, API-13 |
| 通報 → 持ち主が戻らない → 回収依頼 → 正常回収 | AND-02, ADM-05, ADM-07, ADM-08, API-10, API-11, API-13 |
| 通報 → 持ち主が戻らない → 回収依頼 → 現地で自転車なし | AND-02, ADM-05, ADM-07, ADM-09, API-10, API-11, API-13 |
