# Owner Web — 低忠実度ワイヤーフレーム（モバイル優先）

## 目的

QRコードでアクセスする持ち主向けの簡易Webの低忠実度ワイヤーフレーム。目的は速やかに「警告確認 → 仮解除（unlock-temp）→ 本解除（unlock-final）」ができる UX を設計すること。

---

## スクリーン一覧

1. 警告一覧/ステータス画面（Marker Page）
   - 表示要素: 写真（縮小）、警告ステータス（赤）、報告日時、位置、OCRで抜き出した識別情報（可能なら）、説明文、「解除」ボタン
   - 操作: 「解除」ボタンで仮解除APIを呼ぶ

2. 仮解除完了モーダル
   - 表示要素: 仮解除成功メッセージ、`eligibleFinalAt`（15分後）・`expiresAt`（24時間後）、本解除ボタンの有効化状態、閉じる

3. 本解除（最終解除）画面/モーダル
   - 表示要素: 「本解除を実行しますか？」の確認、最終解除ボタン（`eligibleFinalAt` 到達後に有効）、完了メッセージ

4. タイマー画面（省スペースで右上表示）
   - 表示要素: 本解除までの残り時間（15分カウントダウン）、自動解除までの残り時間（24時間）※必要時

5. ヘルプ / FAQ
   - 表示要素: 流れの説明（なぜ一時解除があるか）、悪用時の連絡先、操作履歴の確認方法

6. エラー状態画面
   - QR無効、対象データが見つからない、サーバエラー、レート制限に達した場合のメッセージ

## UI/UX 戦術メモ

- **モバイルファースト**で大きめの主要アクションボタン（目立つ色）を配置。
- **視認性**: 写真とステータスを最上部で強調。色で即時に状態を伝える。
- **操作ガイド**: 仮解除→本解除の意味を簡潔に文で説明。
- **アクセシビリティ**: 色のみでなくテキストラベルを必ず併記（色覚対応）。
- **オフライン/低速回線対策**: 画像は低解像度を先に表示し、詳細は遅延読み込み。
