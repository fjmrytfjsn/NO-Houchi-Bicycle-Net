# クーポン費用試算メモ

**プロジェクト:** NO-Houchi Bicycle Net
**バージョン:** 1.0

## 1. 目的

本メモは、地域協賛型インセンティブとして想定しているクーポンについて、

- 何にコストがかかるか
- どの変数で費用が増減するか
- 実証でどの水準から始めると扱いやすいか

を整理するための補足資料である。

## 2. 仕様前提

現時点でリポジトリ内の既存仕様から読める前提は次のとおり。

| 項目 | 内容 | 参照 |
| --- | --- | --- |
| 発行タイミング | 本解除完了時にクーポンを自動発行 | [backend/README.md](../backend/README.md) |
| 額面 | 500円分のお買い物券または割引クーポン | [backend/README.md](../backend/README.md) |
| 有効期限 | デフォルト30日間 | [backend/README.md](../backend/README.md) |
| スコープ | 試作品では必須受入基準に含めず、将来構想扱い | [docs/owner-api.md](./owner-api.md), [docs/basic-design.md](./basic-design.md) |

このため、今の段階で詰めるべきは「実装費」ではなく、**運用時のクーポン原資と精算・事務費の設計**である。

## 3. クーポン費用の分解

クーポン費用は、次の4つに分けて考えると整理しやすい。

| 区分 | 内容 | 変動/固定 |
| --- | --- | --- |
| 原資負担 | 実際に使われたクーポン額のうち、自治体または運営側が負担する分 | 変動 |
| 精算事務費 | 利用済みクーポンの確認、店舗への精算、問い合わせ対応 | 変動 |
| 店舗運営費 | 協賛店舗の募集、説明、掲載、月次連絡 | 固定 |
| 不正・失効バッファ | 二重利用、期限切れ、キャンペーン調整の吸収枠 | 準固定 |

費用は「発行枚数」ではなく「利用枚数」で見るのが重要である。未使用クーポンは訴求上の効果はあるが、原資負担は発生しない。

## 4. 試算式

### 4.1 基本式

年間費用は次の式で置ける。

```text
年間クーポン費用
= 利用枚数 × (自治体負担額 + 精算事務費)
+ 店舗運営費
+ 不正・失効バッファ
```

さらに、利用枚数は以下で分解できる。

```text
利用枚数
= 本解除件数 × クーポン利用率
```

自治体負担額は以下で置ける。

```text
自治体負担額
= クーポン額面 × 自治体負担率
```

### 4.2 パラメータ定義

| パラメータ | 記号 | 初期値の置き方 |
| --- | --- | --- |
| 本解除件数 | `F` | 実証で観測する年間本解除件数 |
| クーポン利用率 | `R` | 発行枚数に対する実利用割合 |
| 額面 | `V` | 500円で固定 |
| 自治体負担率 | `G` | 0% / 50% / 70% / 100% で比較 |
| 精算事務費 | `S` | 1利用あたり20円を初期仮置き |
| 店舗運営費 | `O` | 年間120,000円を初期仮置き |
| 不正・失効バッファ | `B` | 年間30,000円を初期仮置き |

## 5. 負担モデル別の1利用あたり費用

500円クーポンを前提にすると、自治体側の1利用あたり費用は次のように置ける。

| モデル | 自治体負担率 | 原資負担 | 精算事務費 | 1利用あたり自治体費用 |
| --- | --- | --- | --- | --- |
| 店舗全額負担 | 0% | 0円 | 20円 | 20円 |
| 折半負担 | 50% | 250円 | 20円 | 270円 |
| 推奨初期案 | 70% | 350円 | 20円 | 370円 |
| 自治体全額負担 | 100% | 500円 | 20円 | 520円 |

`20円` の精算事務費は、紙の請求書を前提にせず、簡易な月次集計と振込確認で回す場合の仮置きである。店舗数や精算頻度が増える場合はここが先に膨らむ。

## 7. 実証での推奨運用案

初期実証では、次の条件から始めるのが扱いやすい。

| 項目 | 推奨値 | 理由 |
| --- | --- | --- |
| 額面 | 500円 | 既存 backend 実装と一致し、説明が単純 |
| 自治体負担率 | 70% | 店舗にも参加インセンティブを残しつつ、利用者への魅力も確保しやすい |
| 利用上限 | 1人1回 | 不正抑制と説明簡素化 |
| 有効期限 | 30日 | 既存実装と一致し、短すぎず長すぎない |
| 対象店舗数 | 5〜10店舗 | 精算運用を壊さずに回しやすい |
| 年間固定予算の目安 | 15万〜25万円 | 店舗運営費と少量利用を吸収しやすい |

この条件なら、**年間本解除300件規模でも約21.7万円/年** で回せるため、PoC としては比較的扱いやすい。

## 8. 実証で追うべき指標

クーポン費用は、発行枚数だけ見ても妥当性を判断できない。最低限、次の指標を毎月追うべきである。

| 指標 | 意味 |
| --- | --- |
| 本解除件数 | クーポン発行母数 |
| クーポン利用率 | 実際に原資が発生した割合 |
| 1利用あたり自治体費用 | 負担率と事務費を含んだ実コスト |
| 店舗あたり月間利用件数 | 店舗側の体感価値 |
| クーポン経由の再移動率 | 放置解消行動への寄与度 |

## 9. 次に詰めるべき論点

次の4点が決まると、クーポン費用はかなり現実的になる。

1. 自治体と店舗の負担割合をどこで切るか
2. 利用率を何%で見込むか
3. 精算を月次にするか四半期にするか
4. PoC の年間本解除件数を何件で置くか

現状の仕様に一番素直なのは、**500円・30日・本解除時発行**を維持したまま、まずは `70%負担 / 5〜10店舗 / 年間300件想定` の運用案で行政・商店街双方の納得感を確認する進め方である。
