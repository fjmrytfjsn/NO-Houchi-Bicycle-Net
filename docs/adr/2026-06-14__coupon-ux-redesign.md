# クーポンUXの再設計と目的の再定義

## 日付
2026-06-14

## ステータス
Accepted

## コンテキスト
放置自転車の持ち主がアプリ上でQRコードを読み取って本解除を行った際、これまでクーポンが自動で即時発行される仕様となっていた。
しかし、このままでは「放置した人が簡単にクーポンをもらえてしまう（ご褒美になってしまう）」というモラルハザードの懸念が指摘された。
クーポンの本来の目的は「自転車を貯め集めさせるモチベーション」ではなく、「通報された自転車を今すぐ移動させるための心理的なトリガー（損失回避）」であるべきである。

## 決定事項
クーポンの受取フローを以下のように変更し、フロントエンド（Owner Web）に実装する。

1. **SMS認証（1人あたりの受取制限）**
   - 持ち主が不当に複数回受け取ることを防ぐため、電話番号ベースのSMS認証をキーとする。
   - ※プロトタイプではフロントエンド上のモックUIとして実装する。
2. **ルーレットによるランダム額の決定**
   - 毎回確実にもらえるわけではない（または額がブレる）ランダム性を持たせることで、報酬への「慣れ」を防ぐ。
3. **15分の超短期タイマー（損失回避の活用）**
   - クーポンコード表示後、15分で無効になるカウントダウンタイマーを設置する。
   - レジの直前でしか開けない制約を課すことで、「いつでも使える権利」から「今動かないと失う権利」へと心理的価値を転換し、放置自転車の即時移動を促す。

## 影響範囲
- **Owner Web (UI/UX)**: `SMSAuthPanel`, `CouponRoulette`, `CouponTimer` コンポーネントを新設し、本解除後のフローを改修。
- **バックエンド / データモデル**: 現状はフロントエンドのモックとして実装しているため、即時のスキーマ変更は不要。ただし、将来的に実際のSMS送信機能や月1回制限の履歴管理（電話番号ハッシュ等）を実装する必要がある。

## 備考
本変更により、地域協賛型インセンティブ（クーポン）の位置づけが、単なる「ご褒美」から「行動経済学に基づく即時移動のナッジ（行動誘導）」へと明確に再定義された。
